# GetNumberOfTuples — Tuple 的命名设计

## 背景

用 `xCoords->GetNumberOfTuples()` 获取坐标数组长度，好奇为什么叫 "Tuple" 而不是 "Size" 或 "Count"。

## Tuple 是什么

**Tuple = 一组值（元组）**。

VTK 的数组是**按行组织的结构化数据**，每行是一个物理量，每行有几个分量由 Components 决定：

```
标量: 1-Tuple = [3.14]              → 1 个 Component
向量: 3-Tuple = [1.0, 0.0, 0.5]     → 3 个 Components
张量: 9-Tuple = [m00, m01, ..., m22] → 9 个 Components
```

## 三个关键方法

| 方法 | 返回 | 举例（100点速度场） |
|------|------|---------------------|
| `GetNumberOfTuples()` | 有多少"组"数据 | 100 个点 |
| `GetNumberOfComponents()` | 每组有几个值 | 3（vx, vy, vz） |
| `GetNumberOfValues()` | 总元素数 | 100 × 3 = 300 |

## 为什么叫 Tuple 不是 Size

因为 VTK 的数组不是简单的线性容器。同一套 API 要同时表示标量、矢量、张量数组。用 "Tuple + Component" 的二维模型统一描述，比用 "Size" 更精确。

## 我的理解

> Tuple = 数据库里的一行。每个数据点是一行，每行有几个字段取决于数据类型（标量=1列，矢量=3列）。

这个概念贯穿 VTK 所有数组操作——`InsertNextTuple()`、`GetTuple()` 等。
