# VTK "代码不生效" 四步排查法

## 背景

`buildRectilinearGrid` 缺了 `SetDimensions()` 导致什么都不显示，定位花了不少时间。需要一套系统的排查方法。

## 四步排查

### Step 1: PrintSelf — 数据有没有？

```cpp
std::cout << dataset->GetClassName() << std::endl;
std::cout << "  Points: " << dataset->GetNumberOfPoints() << std::endl;
std::cout << "  Cells:  " << dataset->GetNumberOfCells() << std::endl;
std::cout << "  Bounds: [" << dataset->GetBounds()[0]
          << ", " << dataset->GetBounds()[1] << "]" << std::endl;

auto scalars = dataset->GetPointData()->GetScalars();
std::cout << "  Scalars: " << (scalars ? scalars->GetName() : "NULL") << std::endl;
```

**常见发现**：Cells = 0 → 网格结构没建立。

### Step 2: 管线连接

```
Source ──→ Filter ──→ Mapper ──→ Actor ──→ Renderer ──→ RenderWindow
  ✅         ?           ✅         ✅         ✅            ✅
```

VTK 不会报"忘了连"，但 `mapper->GetInput()` 返回 null 就是没连上。

### Step 3: 相机

```cpp
renderer->ResetCamera();
// 如果 ResetCamera 后还看不到 → 数据坐标可能在异常范围
```

### Step 4: 对比已知正确代码

拿一个能跑的文件，逐行对比。这次 RectilinearGrid 就是缺 SetDimensions，和 ImageData 对比才发现的。

## 防御性习惯

| 习惯 | 防什么 |
|------|--------|
| 数据构造完立刻 `cout` 点数和单元数 | 半小时后才发现没数据 |
| 每次加新类型，复制上一个能跑的完整结构再改 | 少了哪行一眼看出来 |
| 写完 `new` 下一行就写对应的 `Set*` | API 成对调用被遗漏 |
